# BPW 맵 에디터 명세

> **문서 목적:** 관리자 맵 에디터를 **심시티·BuildIt류 제작 UX**에 가깝게 구현하기 위한 요구사항을 정리한다. 스키마·겹침의 정본은 **[DB.md](../DB.md)** 이다.  
> **URL·관리자 라우팅:** **[ADMIN.md](./ADMIN.md)** 참고.

## 1. 범위

| 포함 | 제외 (별도 문서·후속) |
|------|------------------------|
| 타일 브러시, 도로·오브젝트 배치, 부지 직사각형, 건물 배치, 불도저 | 입점 문의 이력 테이블·부지 삭제 시 문의 보존 정책 (미결정) |
| `map_tile` / `map_object` / `map_lot` / `map_building` 편집 | 공개 `/village` 플레이어 동선 변경 없음 |
| 관리자 경로 `/admin/editor` | 일반 사용자에게 에디터 링크 노출 없음 |

## 2. 라우팅

- 진입 경로: **`/admin/editor`** ([ADMIN.md](./ADMIN.md)).
- 인증·권한은 관리자 공통 정책을 따른다.

## 3. 레이어 (그리기 순서: 아래 → 위)

DB 모델과 동일하다 ([DB.md §1.9](../DB.md)).

| 순서 | 의미 | 테이블 |
|------|------|--------|
| 1 | **지면** | `map_tile` + `tile_type` |
| 2 | **배치층** — 같은 시각적 높이. **한 타일에는 오브젝트 점유와 부지 영역이 동시에 있으면 안 됨** | `map_object` **또는** `map_lot` |
| 3 | **건물** | `map_building` (`lot_id`로 부지와 연결) |

- **도로**는 지면 타일이 아니라 **`map_object`** 로 둔다. 불도저로 도로만 제거하면 해당 칸의 **`map_tile`(잔디 등)은 유지**된다 ([DB.md §1.10](../DB.md)).

## 4. 겹침·불도저 (제품 규칙)

- **지면:** 칸당 하나의 `tile_key`.
- **두 `map_object`:** 점유 타일이 겹치면 안 됨.
- **두 `map_lot`:** `area` 직사각형이 겹치면 안 됨.
- **오브젝트 ↔ 부지:** 한 칸에 동시에 속하면 안 됨 → 한쪽을 **불도저로 제거**한 뒤 다른 쪽을 배치.
- **건물:** 다른 건물의 `area`와 겹치면 안 됨 (가장 엄격).
- **부지 불도저:** 해당 **`map_lot` 행 전체** 삭제(또는 동등한 해제). 영역은 한 행의 `area`로 정의되어 있다 ([DB.md §2.7](../DB.md)).

문의 데이터와 CASCADE 동작은 현재 스키마를 따르며, **보존 정책은 추후 결정**한다.

## 5. 툴·UX

에디터 **툴 모드**(브러시 vs 스탬프 등)는 **클라이언트 상태**로 두고, 필요 시 나중에 `object_type.meta` 등으로 서버 힌트를 추가할 수 있다 ([DB.md](../DB.md) 논의와 동일).

| 툴 | 동작 개요 | 저장 대상 |
|----|-----------|-----------|
| **타일 브러시** | 지면 타입을 슬라이딩으로 칠함 | `map_tile` upsert |
| **도로** | 오브젝트 중 도로 타입 — **연속 배치(브러시)** 허용 (예외) | `map_object` |
| **기타 오브젝트** | 기본은 클릭·스탬프 배치 | `map_object` |
| **부지** | 직사각형 지정 후 확정 | `map_lot` |
| **건물** | 부지와 연결하여 배치 — **드래그 앤 드롭에 가깝게** | `map_building` |
| **불도저** | 대상 레이어에 따라 오브젝트만 / 부지 단위 / 건물 등 | 해당 행 삭제 또는 지면 유지 |

## 6. 데이터 읽기·쓰기

- **읽기:** 기존 `loadWorldMapBundle`(slug 등)으로 맵 번들 로드; 에디터는 동일 데이터를 편집 가능 상태로 표시.
- **쓰기:** 변경분을 Supabase에 반영 (클라이언트 직접 또는 서버 액션). **관리자만** 쓰기 가능하도록 RLS·역할을 맞춘다.
- **`lot_phase`:** 입점 워크플로용 캐시 ([DB.md §3](../DB.md)). 에디터 필수 경로에서 직접 수정할 필요는 없다.

## 7. 구현 순서 (권장)

1. `/admin` 레이아웃·인증 가드 ([ADMIN.md](./ADMIN.md)).
2. `/admin/editor` 페이지에 Phaser(또는 기존 월드 런타임 재사용) + 그리드·카메라.
3. 툴바: 타일 브러시 → 도로 브러시 → 오브젝트 스탬프 → 부지 → 건물 → 불도저 순으로 연결.
4. 저장: 로컬 프리뷰 후 배치 단위 또는 주기적 persistence.
