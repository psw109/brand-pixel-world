# Brand Pixel World (BPW) — 프로젝트 설계 개요

> **문서 목적:** 웹 기반 픽셀 마을 탐험 + 브랜드 건물 상호작용 서비스의 전체 그림을 한곳에 두고, 개발·기획이 바뀔 때마다 이 파일을 수정하며 맞춰 나간다.

## 1. 프로젝트 한 줄 정의

**2D 탑다운 픽셀 마을을 드래그(또는 터치)로 둘러보고, 입점 가능 부지를 표시하며, 부지 또는 건물을 탭해 입점 문의·브랜드 경험(모달·임베드·외부 링크 등)으로 이어지는 웹 서비스.**

- 설치 없이 브라우저에서 동작 (웹 + 모바일 브라우저 고려).
- 세부 범위는 [§3](#3-현-단계-범위와-확장-여지), 데이터·맵 규칙은 **[DB.md](./DB.md)** 가 정본이다.

## 2. 이 프로젝트가 하는 일 (사용자 관점)

| 단계            | 사용자 행동                   | 기대 결과                                                              |
| --------------- | ----------------------------- | ---------------------------------------------------------------------- |
| 진입            | 홈(`/`) 또는 마을(`/village`) | 픽셀 마을(월드) 또는 안내 화면                                         |
| 탐색            | 마우스 드래그 / 터치 드래그   | 마을을 이동하며 **부지**·건물·타일 탐색                                |
| 상호작용 (부지) | 비어 있는 입점 부지 영역 탭   | 입점·홍보 문의 UI (향후: 폼 → Supabase `lot_inquiry` 등)               |
| 상호작용 (건물) | 건물 footprint 탭             | 건물별 모달·콘텐츠 (`map_building.modal` 계약은 [DB.md §2.9](./DB.md)) |

### 2.1 부지·건물 데이터

- 맵·부지·건물·타일·오브젝트의 **저장 형식·겹침·레이어**는 **[DB.md](./DB.md)** 를 따른다 (PostgreSQL, `map` / `map_tile` / `map_object` / `map_lot` / `map_building` 등).
- 부지는 **직사각형 `area`(JSON)** 로 표현한다. 클라이언트 타입은 `loadWorldMap` 번들과 [`worldMapTypes`](../src/lib/map/worldMapTypes.ts) 등을 참고한다.

### 2.2 입점 문의·운영

- 스키마상 접수는 **`lot_inquiry`** (부지 `lot_id` 연결). 예전 초안 문서의 `leads` 단일 테이블과는 다르다.
- **관리자:** `/admin/inquiries` 계획 ([dev/ADMIN.md](./dev/ADMIN.md)). 인증·RLS는 구현 단계에서 정한다.

## 3. 현 단계 범위와 확장 여지

### 3.1 구현된 것 (요약)

- **Next.js App Router** 홈(`/`), **`/village`** 에서 Phaser 월드 ([`VillageGame`](../src/components/game/VillageGame.tsx) 등).
- Supabase에서 **`loadWorldMapBundle`** 으로 맵 메타·타일·부지·건물·오브젝트 로드 (slug 기본 `main`).
- **`map.tile_px`** 필수(정사각 타일 한 변 px). DB·클라 검증 일치.

### 3.2 진행 중·예정

- **맵 에디터:** [`dev/MAP_EDITOR.md`](./dev/MAP_EDITOR.md) (심시티식 툴·레이어).
- 입점 폼·관리자 문의 목록·건물 모달 고도화.

### 3.3 나중에 붙여도 되는 것

- 줌, 청크 스트리밍, 광고주 대시보드, 실시간 다국어 등.

## 4. 화면 구조: UI 레이어 vs 맵 데이터 레이어

**게임/맵 데이터의 레이어**(지면 → 배치층 → 건물)는 **[DB.md §1.9](./DB.md)** 와 동일하다.

**UI 관점**에서는 여전히 다음 두 층으로 나누어 설계한다.

| 겹침 순서 | 이름                | 역할                                    |
| --------- | ------------------- | --------------------------------------- |
| 아래      | **월드(Phaser)**    | 타일·부지·오브젝트·건물 렌더, 패닝 입력 |
| 위        | **오버레이(React)** | 탭 후 모달·폼·iframe 등 DOM UI          |

모바일에서 스크롤·뒤로가기가 월드와 오버레이에 동시에 먹히지 않도록 포커스·스크롤 잠금 정책을 둔다.

## 6. 기술 스택 (현재 레포 기준)

| 영역            | 선택                                                                                   | 비고                   |
| --------------- | -------------------------------------------------------------------------------------- | ---------------------- |
| 프론트          | **Next.js 16** (App Router) + **TypeScript**                                           | `package.json` 기준    |
| UI              | **React 19**, **Tailwind CSS 4**                                                       |                        |
| 검증·도메인     | **Zod**                                                                                | 공개 env 등            |
| 월드 렌더링     | **Phaser 3**                                                                           | 캔버스·패닝·스프라이트 |
| 클라이언트 상태 | **Zustand**                                                                            |                        |
| 백엔드·DB       | **Supabase** (Postgres, RLS, 마이그레이션 정본은 `supabase/migrations/`)               |                        |
| 배포            | **Vercel**                                                                             |                        |
| 관리자          | **`/admin/**`\*\* 계획 ([dev/ADMIN.md](./dev/ADMIN.md)), 인증 미구현 시 접근 통제 필수 |

## 7. 용어 짧게

| 용어             | 의미                                                                   |
| ---------------- | ---------------------------------------------------------------------- |
| 월드 / 마을      | 사용자가 패닝하는 2D 맵 전체                                           |
| 타일 / 지면      | `map_tile`·`tile_type`                                                 |
| 배치층           | `map_object` 또는 `map_lot` (같은 칸에 동시 불가) — [DB.md](./DB.md)   |
| footprint / area | 직사각형 영역 JSON (`tile_x`, `tile_y`, `width_tiles`, `height_tiles`) |
| 부지             | `map_lot`                                                              |
| 건물             | `map_building`                                                         |
| 오버레이         | React DOM으로 월드 위에 띄우는 UI                                      |
| 입점 문의        | `lot_inquiry`                                                          |
